# CS210-Corner-Grocer
I developed this project for the CS 210 course at Southern New Hampshire University. The objective was to design a C++ application for the Corner Grocer that analyzes daily transaction files containing grocery items purchased. The program reads the input file, counts the frequency of each item, stores the data, generates a backup frequency file, and provides users with a menu to search for item frequencies or display reports.

This project addresses the challenge of manually tracking inventory trends by automating the counting process and presenting information in an organized, accessible format. As a result, the grocery store can make more informed decisions regarding product placement and inventory management.

What I Did Well

A key strength of this project was the application of object-oriented programming principles. I divided the program into multiple source and header files, which improved code readability, maintainability, and debugging. Additionally, I implemented distinct functions for each major task, including reading the input file, computing item frequencies, printing all frequencies, generating a histogram, and writing the data to a backup file.

Another notable strength was the development of a user-friendly menu system that validates user input and enables users to perform various operations repeatedly until they choose to exit the program.

Opportunities for Improvement

If I were to continue developing this project, I would enhance the program by implementing more robust error handling and additional input validation. For instance, making item searches case-insensitive would ensure users receive accurate results regardless of capitalization.

I would also consider incorporating advanced C++ features, such as exception handling, templates, and the Standard Template Library (STL), to improve efficiency and reduce code duplication. Additional enhancements, such as sorting items by frequency, exporting reports in various formats, or integrating a graphical user interface, would further strengthen the program.

Challenges and How I Overcame Them

The most challenging aspect of this project was managing file input and output and accurately storing item frequencies in a map. It required significant effort to understand how to read each line of the input file, update frequency counts, and output results in multiple formats.

I addressed these challenges by dividing the project into smaller tasks, testing each function individually, and utilizing Visual Studio's debugging tools to identify and resolve errors. Additionally, I relied on course resources, class materials, C++ documentation, and instructor feedback to deepen my understanding.

Maintainability, Readability, and Adaptability

I designed the program for maintainability by separating the code into multiple files and assigning each function a specific responsibility. The use of meaningful variable names, consistent formatting, and comprehensive comments throughout the code enhances readability and facilitates understanding for other developers.

The modular design also increases the application's adaptability. New menu options or additional reporting features can be incorporated without significant changes to the existing code structure, thereby simplifying future enhancements.
